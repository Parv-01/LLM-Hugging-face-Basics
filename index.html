§ 1 — Foundation

Phase 0: Mathematical Foundations

This is the hardest phase to stay motivated through, and the most important phase of all. Every single AI concept you will ever learn is written in this language.

Phase 0

Mathematics for Machine Learning

⏱ Duration: 2–4 months · Daily: 2–3 hours

You do not need a PhD in mathematics. You need deep working knowledge of four areas: Linear Algebra, Calculus, Probability, and Optimization. Everything else is notation on top of these.

0.1 — Linear Algebra

Everything in machine learning is a matrix operation. Images are matrices of pixel values. Text is vectors of numbers. Neural networks are stacked matrix multiplications. If you do not understand linear algebra deeply, you will be confused about every model you ever use.

// Core objects you must master: Vector: v ∈ ℝⁿ — a column of n real numbers Matrix: A ∈ ℝᵐˣⁿ — m rows, n columns Matrix Mul: (AB)ᵢⱼ = Σₖ Aᵢₖ · Bₖⱼ // The two most important decompositions in ML: Eigendecomposition: A · v = λ · v where v = eigenvector, λ = eigenvalue Intuition: v is a direction A only SCALES, not rotates. SVD (Singular Value Decomposition): A = U · Σ · Vᵀ U = left singular vectors (ℝᵐˣᵐ, orthogonal) Σ = diagonal singular values (ℝᵐˣⁿ) Vᵀ = right singular vectors (ℝⁿˣⁿ, orthogonal) WHY IT MATTERS: → PCA (dimensionality reduction) IS just SVD → Word embeddings reduce via SVD → Attention weights have low-rank structure Dot Product & Cosine Similarity: a · b = |a| · |b| · cos(θ) similarity(a,b) = (a · b) / (|a| · |b|) → This is literally how attention scores are computed!

Vector Spaces

What makes a valid vector space. Span, basis, linear independence. You must be able to prove these formally.

Matrix Operations

Transpose, inverse, rank, determinant, trace. Know when a matrix is invertible and why it matters.

Norms

L1 norm (||x||₁ = Σ|xᵢ|), L2 norm (||x||₂ = √Σxᵢ²). These become regularization terms in ML.

Eigenvalues/Eigenvectors

Solve Av = λv. Understand the characteristic polynomial. Connect to PCA and covariance matrices.

SVD Decomposition

The most important matrix factorization in all of ML. Drives dimensionality reduction and latent semantic analysis.

Projections

Projection of v onto subspace W. This is the geometric foundation of attention and regression both.

0.2 — Calculus & Gradients

Neural networks learn by computing how much each weight contributed to the error, and nudging it in the right direction. This "how much" is the gradient. Calculus is the engine of learning.

// The derivative: rate of change df/dx = lim[h→0] (f(x+h) - f(x)) / h // Partial derivative: change w.r.t. ONE variable ∂f/∂xᵢ (hold all other variables constant) // Gradient: ALL partial derivatives stacked as a vector ∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ Intuition: gradient points in direction of STEEPEST ASCENT // Chain Rule — THE most important rule in deep learning If y = f(g(x)), then: dy/dx = (dy/dg) · (dg/dx) Example: Loss = CrossEntropy(Softmax(W·x + b)) ∂Loss/∂W = ∂Loss/∂softmax · ∂softmax/∂(Wx+b) · ∂(Wx+b)/∂W → Backpropagation is JUST repeated chain rule! // Jacobian: matrix of all partial derivatives (vector → vector) J[i,j] = ∂fᵢ/∂xⱼ (needed for batch ops in NN layers) // Hessian: second-order derivatives (curvature of loss surface) H[i,j] = ∂²f / (∂xᵢ ∂xⱼ) → Used in second-order optimizers (Newton's method, K-FAC)

0.3 — Probability & Statistics

Language models are probability distributions over text. Every "token prediction" is a categorical probability. You cannot deeply understand LLMs without being fluent in probability theory.

// Basic Rules P(A ∪ B) = P(A) + P(B) - P(A ∩ B) P(A | B) = P(A ∩ B) / P(B) // Bayes' Theorem — the foundation of probabilistic ML P(θ | data) = P(data | θ) · P(θ) / P(data) Posterior = Likelihood × Prior / Evidence ML Translation: θ = model parameters P(θ) = prior belief about parameters P(data|θ) = likelihood: how well θ explains data P(θ|data) = posterior: updated belief after seeing data // Maximum Likelihood Estimation (MLE) — how models train θ* = argmax_θ Σᵢ log P(xᵢ | θ) → Training a neural network = maximizing log-likelihood → Cross-entropy loss = negative log-likelihood! // Key Distributions you must know: Bernoulli: P(X=1) = p, P(X=0) = 1-p (binary classification) Categorical: P(X=k) = pₖ (next-token prediction!) Gaussian: P(x) = (1/√2πσ²) exp(-(x-μ)²/2σ²) (weight initialization) KL Divergence: D_KL(P||Q) = Σ P(x) log(P(x)/Q(x)) (used in RLHF, VAEs) // Entropy — measures uncertainty H(P) = -Σ P(x) log P(x) → A model with HIGH perplexity = HIGH entropy = very uncertain → Cross-entropy loss = H(P_true, P_model) = -Σ y log ŷ

0.4 — Optimization Theory

Training any model is an optimization problem: minimize the loss function over the parameter space. Understanding what makes optimization hard — and how algorithms overcome it — is essential for research.

// Gradient Descent: the core update rule θ ← θ - η · ∇_θ L(θ) where η = learning rate, L = loss, θ = parameters // Variants: SGD (Stochastic): θ ← θ - η · ∇L(xᵢ) (one sample) Mini-batch SGD: θ ← θ - η · ∇L(x_{1..B}) (B samples) // Momentum — escape flat regions v ← β·v + (1-β)·∇L(θ) θ ← θ - η·v // Adam Optimizer — most popular in LLM training m ← β₁·m + (1-β₁)·g (1st moment: mean) v ← β₂·v + (1-β₂)·g² (2nd moment: variance) m̂ = m/(1-β₁ᵗ), v̂ = v/(1-β₂ᵗ) (bias correction) θ ← θ - η · m̂ / (√v̂ + ε) Typical values: β₁=0.9, β₂=0.999, ε=1e-8 // Learning Rate Scheduling Warmup: lr linearly increases for first T_warm steps Cosine: η(t) = η_min + ½(η_max - η_min)(1 + cos(πt/T)) → GPT-3, LLaMA all use cosine decay with warmup

§ 2 — Machine Learning Theory

Phase 1: Classical Machine Learning

Now we use the mathematics. Classical ML builds the intuition and theoretical backbone that you will rely on even when working with the largest LLMs.

Phase 1

Statistical Learning Theory & Classical ML

⏱ Duration: 2–3 months · Daily: 2 hours

You must derive every model from first principles. No black boxes. When you understand logistic regression deeply, you understand the output layer of every LLM.

1.1 — Linear Regression (Derive Everything)

// Model ŷ = Xθ where X ∈ ℝⁿˣᵈ, θ ∈ ℝᵈ, ŷ ∈ ℝⁿ // Loss Function: Mean Squared Error L(θ) = (1/n) ||y - Xθ||²₂ = (1/n)(y - Xθ)ᵀ(y - Xθ) // Closed-form solution (Normal Equations) ∇_θ L = -(2/n) Xᵀ(y - Xθ) = 0 ⟹ θ* = (XᵀX)⁻¹ Xᵀy // Probabilistic interpretation (this connects to LLMs!) Assume: y = Xθ + ε, ε ~ N(0, σ²I) Then: P(y|X,θ) = N(Xθ, σ²I) MLE of θ: argmax_θ log P(y|X,θ) = argmin_θ ||y - Xθ||² → MSE loss IS negative log-likelihood under Gaussian noise! // Gradient Descent derivation ∇_θ L = -(2/n) Xᵀ(y - Xθ) θ ← θ - η · (-(2/n) Xᵀ(y - Xθ)) θ ← θ + (2η/n) Xᵀ(y - Xθ)

1.2 — Logistic Regression & Classification

// Sigmoid: squashes ℝ → (0,1) — probability output σ(z) = 1 / (1 + e⁻ᶻ) σ'(z) = σ(z)(1 - σ(z)) ← you must derive this! // Model: P(y=1 | x) = σ(θᵀx) // Binary Cross-Entropy Loss L(θ) = -(1/n) Σᵢ [yᵢ log ŷᵢ + (1-yᵢ) log(1-ŷᵢ)] // This IS negative log-likelihood of Bernoulli: P(y|x,θ) = ŷʸ · (1-ŷ)¹⁻ʸ log P = y log ŷ + (1-y) log(1-ŷ) // Softmax — multiclass generalization (used in EVERY LLM output layer) softmax(z)ₖ = e^zₖ / Σⱼ e^zⱼ Categorical cross-entropy: L = -Σₖ yₖ log ŷₖ → When an LLM predicts the next token, it runs softmax on logits

1.3 — Bias-Variance Tradeoff & Regularization

// Decomposition of Expected Error E[(y - ŷ)²] = Bias²(ŷ) + Var(ŷ) + σ²_noise Bias²: Error from wrong model assumptions (underfitting) Variance: Error from sensitivity to training data (overfitting) Noise: Irreducible error // Regularization: add penalty to loss to reduce variance L2 (Ridge): L_reg = L + λ||θ||²₂ → shrinks weights toward 0 L1 (Lasso): L_reg = L + λ||θ||₁ → forces some weights to EXACTLY 0 // Why this connects to Bayesian thinking: L2 regularization = MAP estimation with Gaussian prior on θ L1 regularization = MAP estimation with Laplace prior on θ // In deep learning: Dropout During training: randomly zero out each neuron with prob p h̃ = h ⊙ mask, mask_i ~ Bernoulli(1-p) Intuition: forces network to learn redundant representations

§ 3 — Deep Learning

Phase 2: Neural Networks & Deep Learning

Here we go deeper. You will build neural networks from scratch and derive backpropagation by hand — multiple times — until it is second nature.

Phase 2

Deep Learning — Theory First, Code Second

⏱ Duration: 3–4 months · Daily: 2–3 hours + coding

Do NOT start with PyTorch tutorials. First, build a neural network in plain NumPy. Then build the backward pass by hand. Then, and only then, move to frameworks.

2.1 — Neural Networks from Scratch

// A single neuron z = wᵀx + b, a = σ(z) // A layer of neurons (vectorized) Z⁽ˡ⁾ = W⁽ˡ⁾ A⁽ˡ⁻¹⁾ + b⁽ˡ⁾ (pre-activation) A⁽ˡ⁾ = g⁽ˡ⁾(Z⁽ˡ⁾) (post-activation) where g is the activation function. // Common Activation Functions: ReLU: g(z) = max(0, z), g'(z) = 1 if z>0 else 0 GeLU: g(z) = z·Φ(z) (Φ = CDF of standard normal) ← Used in GPT! Sigmoid: g(z) = 1/(1+e⁻ᶻ), g'(z) = g(z)(1-g(z)) Tanh: g(z) = (e^z-e⁻ᶻ)/(e^z+e⁻ᶻ) SwiGLU: g(x,y) = x·σ(y) (Swish-Gated Linear Unit) ← Used in LLaMA! // Universal Approximation Theorem (conceptual) A 2-layer NN with enough neurons can approximate ANY continuous function on a compact domain. → This is WHY neural networks work.

2.2 — Backpropagation (Derive It Fully)

// Compute gradient of Loss w.r.t. every weight — via Chain Rule // Define delta (error signal) at each layer: δ⁽ᴸ⁾ = ∇_A L ⊙ g'(Z⁽ᴸ⁾) (output layer) δ⁽ˡ⁾ = (W⁽ˡ⁺¹⁾ᵀ δ⁽ˡ⁺¹⁾) ⊙ g'(Z⁽ˡ⁾) (hidden layers, propagate back) // Gradients for weights and biases: ∂L/∂W⁽ˡ⁾ = δ⁽ˡ⁾ · A⁽ˡ⁻¹⁾ᵀ ∂L/∂b⁽ˡ⁾ = δ⁽ˡ⁾ // The 4 equations of backprop (Rumelhart, 1986): BP1: δᴸ = ∇_a L ⊙ σ'(zᴸ) BP2: δˡ = ((Wˡ⁺¹)ᵀ δˡ⁺¹) ⊙ σ'(zˡ) BP3: ∂L/∂bˡ = δˡ BP4: ∂L/∂Wˡ = δˡ (aˡ⁻¹)ᵀ // Vanishing gradient problem In deep networks: δˡ = δᴸ · Πₖ₌ₗ₊₁ᴸ (Wᵏᵀ · diag(σ'(zᵏ))) If σ' < 1 repeatedly: gradient → 0 (vanishing) If σ' > 1 repeatedly: gradient → ∞ (exploding) Solutions: ReLU activations, BatchNorm, ResNets, gradient clipping

2.3 — Normalization, Residual Connections, and Training Tricks

// Batch Normalization μ_B = (1/m) Σᵢ xᵢ, σ²_B = (1/m) Σᵢ (xᵢ - μ_B)² x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε) yᵢ = γ x̂ᵢ + β (learnable scale and shift) Effect: stabilizes training, allows higher learning rates // Layer Normalization (used in Transformers — NOT batch norm) LN(x) = γ · (x - μ_layer) / √(σ²_layer + ε) + β Difference: normalize ACROSS features, not across batch → Works better for variable-length sequences (like text!) // Residual (Skip) Connections — from ResNet (He et al., 2016) H(x) = F(x) + x Where F(x) = what the layers learn, x = shortcut Effect: gradient flows directly through skip connection → Transformers use this between EVERY sub-layer → Enables training 100s of layers without vanishing gradients // Dropout ỹ = y ⊙ m / (1-p), mᵢ ~ Bernoulli(1-p) (divide by (1-p) to keep expected value same at test time)

2.4 — CNNs and RNNs (Know the History)

Convolutional networks and recurrent networks dominated the decade before Transformers. You need to understand them because (1) Transformers replaced them for specific reasons, and (2) CNNs are still used in vision models that connect to multimodal LLMs.

// Convolution operation (f * g)[n] = Σₖ f[k] · g[n-k] Output[i,j] = Σₘ Σₙ Input[i+m, j+n] · Kernel[m,n] Parameter sharing: same kernel slides over entire image Translation equivariance: shift input → shift output // RNN: hidden state carries information through time hₜ = tanh(Wₓₓ xₜ + Wₕₕ hₜ₋₁ + b) Problem: gradient of L w.r.t. h₁ = Π (∂hₜ/∂hₜ₋₁) → vanishing/exploding gradients over long sequences! // LSTM: gates control information flow fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) (forget gate) iₜ = σ(Wᵢ·[hₜ₋₁, xₜ] + bᵢ) (input gate) c̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) (candidate cell) cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ (cell state update) oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) (output gate) hₜ = oₜ ⊙ tanh(cₜ) WHY TRANSFORMERS REPLACED RNNs: → RNNs must process tokens SEQUENTIALLY — not parallelizable → Transformers process ALL tokens simultaneously → Attention has direct path between any two positions → Training is orders of magnitude faster on GPUs

§ 4 — Natural Language Processing

Phase 3: NLP Foundations

How do we turn human language — messy, ambiguous, context-dependent — into mathematics? This phase answers that question from first principles.

Phase 3

Natural Language Processing — Vectors, Embeddings, Language Models

⏱ Duration: 2 months · Daily: 2 hours

We start with the simplest representation of text and build up to the embeddings that power modern LLMs. Understanding the evolution is key to understanding Transformers.

3.1 — Text Representation

// Step 1: Tokenization "Hello world" → ["Hello", "world"] or ["Hell", "o", "world"] Modern LLMs use Byte Pair Encoding (BPE): - Start with characters - Iteratively merge most frequent adjacent pairs - Vocabulary size: typically 32k–128k tokens // Step 2: One-Hot Encoding (naive baseline) Vocab = {cat, dog, fish}, V = 3 "cat" → [1, 0, 0] "dog" → [0, 1, 0] Problems: - dimensionality = vocab size (tens of thousands) - all words equally "distant" — no semantic info - dot product of any two different words = 0 // Step 3: TF-IDF (for retrieval, still used in RAG systems) TF(t,d) = count(t in d) / |d| IDF(t) = log(N / df(t)) TF-IDF(t,d) = TF(t,d) · IDF(t) Intuition: weight terms that are frequent in doc but rare overall

3.2 — Word Embeddings (The Breakthrough)

// Key insight: represent words as DENSE low-dimensional vectors Embedding dimension d ≪ vocab size V Each word → vector in ℝᵈ (d typically 100–300) // Word2Vec: Mikolov et al. 2013 — two variants Skip-gram: given center word, predict context words Objective: max Σ_t Σ_{-c≤j≤c, j≠0} log P(wₜ₊ⱼ | wₜ) P(wₒ|wᵢ) = exp(vₒ · vᵢ) / Σⱼ exp(vⱼ · vᵢ) (softmax over vocab) CBOW: given context words, predict center word // Remarkable properties learned: vec("King") - vec("Man") + vec("Woman") ≈ vec("Queen") vec("Paris") - vec("France") + vec("Germany") ≈ vec("Berlin") → Linear algebra in embedding space encodes SEMANTIC STRUCTURE // GloVe: Global Vectors (Pennington et al. 2014) Train on co-occurrence matrix Xᵢⱼ = count(word i, word j together) Objective: Σᵢⱼ f(Xᵢⱼ)(vᵢᵀṽⱼ + bᵢ + b̃ⱼ - log Xᵢⱼ)² → Explicit factorization of co-occurrence statistics // Key limitation of both: "bank" near "river" vs "bank" near "money" → same vector regardless of context! → This is why we need CONTEXTUAL embeddings (BERT, GPT)

3.3 — Language Modeling (n-gram → Neural)

// A language model assigns probability to sequences of words P(w₁, w₂, ..., wₙ) = Π P(wₜ | w₁, ..., wₜ₋₁) // n-gram model (classical) Markov assumption: P(wₜ | w₁..wₜ₋₁) ≈ P(wₜ | wₜ₋ₙ₊₁..wₜ₋₁) Bigram: P(wₜ | wₜ₋₁) = count(wₜ₋₁, wₜ) / count(wₜ₋₁) Sparsity problem: most n-grams never seen in training data // Neural Language Model: Bengio et al. 2003 - Look up embeddings for context words - Feed through MLP - Output: softmax over vocabulary P(wₜ | w₁..wₜ₋₁) = softmax(W₂ tanh(W₁[eₜ₋₁;...;eₜ₋ₙ] + b₁) + b₂) This was the precursor to Transformers! // Perplexity — the standard evaluation metric for LMs PPL(W) = exp(-(1/N) Σₜ log P(wₜ | w₁..wₜ₋₁)) = exp(cross-entropy loss) Lower perplexity = better model (less surprised by held-out text) GPT-2: PPL ≈ 35 on WikiText-103 GPT-3: PPL ≈ 20 on WikiText-103 LLaMA-3 70B: PPL ≈ 10

§ 5 — The Architecture That Changed Everything

Phase 4: Transformers — Deep Dive

This is the most important architecture in modern AI. You must understand every equation, every design decision, and every trade-off. There is no shortcut here.

Phase 4

Transformers: Attention Is All You Need

⏱ Duration: 2–3 months · Daily: 3 hours

Read the original paper (Vaswani et al. 2017) ten times. Build it from scratch in PyTorch without looking at existing implementations. This is a rite of passage.

4.1 — The Attention Mechanism

// Intuition: for each word, attend to ALL other words and aggregate // Three learned projections from input X ∈ ℝⁿˣᵈ: Q = X · Wq (queries — "what am I looking for?") K = X · Wk (keys — "what do I contain?") V = X · Wv (values — "what do I communicate?") Wq, Wk ∈ ℝᵈˣᵈₖ, Wv ∈ ℝᵈˣᵈᵥ // Scaled Dot-Product Attention Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V Step-by-step: 1. S = QKᵀ Sᵢⱼ = how much token i attends to token j [n×n] 2. S̃ = S / √dₖ Scale to prevent softmax saturation 3. A = softmax(S̃) Normalize so rows sum to 1 [n×n] 4. O = A · V Weighted sum of values [n×dᵥ] // WHY √dₖ scaling? If Q,K have unit variance, QKᵀ has variance dₖ. Without scaling, softmax saturates (near 0 or 1 everywhere). Dividing by √dₖ normalizes variance back to ~1. // Causal (Masked) Attention — for autoregressive LMs Apply mask before softmax: S̃ᵢⱼ = -∞ for j > i (token i cannot see future tokens) After softmax: Aᵢⱼ = 0 for j > i → lower triangular attention

4.2 — Multi-Head Attention

// Intuition: run h attention "heads" in parallel — each learns different relationships // Example: head 1 = syntax, head 2 = coreference, head 3 = semantics headᵢ = Attention(Q·Wqᵢ, K·Wkᵢ, V·Wvᵢ) MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) · Wₒ where Wqᵢ, Wkᵢ ∈ ℝᵈˣ⁽ᵈ/ʰ⁾ Total parameters in MHA: 4 × d² (for Wq, Wk, Wv, Wo) // Grouped Query Attention (GQA) — used in LLaMA 2/3 Instead of h separate K,V heads, use G groups (G < h) Heads in same group SHARE K,V projections Saves memory during inference (smaller KV-cache) GQA(Q,K,V): h query heads, G key-value heads, h/G per group // Flash Attention (Dao et al. 2022) — critical for efficiency Problem: QKᵀ matrix is O(n²) in memory FlashAttention: tile the computation, never materialize full n×n matrix Result: O(n) memory, same output, 2-4x faster → Used in ALL modern LLM training and inference

4.3 — Full Transformer Architecture

// One Transformer ENCODER block (as in BERT): h¹ = LayerNorm(x + MultiHeadAttn(x, x, x)) (self-attention + residual) h² = LayerNorm(h¹ + FFN(h¹)) (FFN + residual) // Feed-Forward Network (FFN) in each block: FFN(x) = W₂ · ReLU(W₁x + b₁) + b₂ Dimensions: d → 4d → d (expands then contracts) Parameters: 2 × d × 4d = 8d² (LARGER than attention!) // Positional Encoding (original sin+cos version): PE(pos, 2i) = sin(pos / 10000^(2i/d)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d)) Modern alternative: RoPE (Rotary Position Embeddings — Su et al. 2021) Apply rotation matrix R(mθ) to Q,K before attention: qₘᵀ kₙ = (Rₘ qₘ)ᵀ (Rₙ kₙ) = qₘᵀ Rₙ₋ₘ kₙ → Depends only on RELATIVE position (m-n), not absolute! → Used in LLaMA, Falcon, Mistral, Qwen, and most modern LLMs // Full Parameter Count (GPT-2 Small: d=768, L=12 layers, h=12 heads): Embedding: V × d = 50257 × 768 = 38.6M Per layer: 4d² (attn) + 8d² (FFN) = 12d² = ~7.1M 12 layers: 12 × 7.1M = ~85M Total: ~117M params ← matches official GPT-2 small count!

4.4 — Training Objectives

// GPT-style: Causal Language Modeling (CLM) L_CLM = -(1/T) Σₜ log P(wₜ | w₁, w₂, ..., wₜ₋₁; θ) Autoregressive: predict each token from left context only. Used by: GPT-2, GPT-3, GPT-4, LLaMA, Mistral, Gemma, Qwen, etc. // BERT-style: Masked Language Modeling (MLM) Randomly mask 15% of tokens, predict them: L_MLM = -Σ_{i ∈ masked} log P(wᵢ | w̃; θ) w̃ = sequence with some tokens replaced by [MASK] Bidirectional context — sees left AND right. Used by: BERT, RoBERTa, ELECTRA, DeBERTa // T5-style: Span Corruption Replace contiguous spans with single [MASK] tokens: Model must recover entire spans. Used by: T5, FLAN-T5, mT5 // Modern trend: multimodal pre-training LLaVA, GPT-4V: L = L_text + L_image_text_alignment CLIP: Contrastive loss between image-text pairs: L_CLIP = -log [exp(sim(vᵢ,tᵢ)/τ) / Σⱼ exp(sim(vᵢ,tⱼ)/τ)]

Phase 5

5.1 — Scaling Laws

// Kaplan et al. 2020 (OpenAI) — original scaling laws Loss scales as power law with compute, data, and parameters: L(N) = (Nc/N)^αN (w.r.t. parameters N) L(D) = (Dc/D)^αD (w.r.t. dataset size D) L(C) ≈ C^(-0.05) (w.r.t. compute C in FLOPs) Optimal allocation: N ∝ C^0.73, D ∝ C^0.27 → More compute → increase params FASTER than data // Chinchilla (Hoffmann et al. 2022 — DeepMind) — revised laws For compute-optimal training: N_opt = D_opt (parameters and tokens should scale EQUALLY) C ≈ 6ND FLOPs (6 flops per parameter per token — forward + backward) Chinchilla-optimal: 1B param model → train on 20B tokens GPT-3 (175B params, 300B tokens) was UNDER-trained by Chinchilla standards! LLaMA: 65B params × 1.4T tokens → trained far beyond Chinchilla optimal → better inference-time performance // Emergent Abilities (Wei et al. 2022) Some capabilities appear ONLY above critical parameter count: - Chain-of-thought reasoning: emerges ~100B params - BIG-Bench tasks: sudden jumps at scale thresholds Debate: are these truly emergent or artifact of metrics?

5.2 — Fine-Tuning & Alignment

// Full Fine-Tuning: update ALL parameters on task data Problem: requires same memory as pre-training, expensive // LoRA: Low-Rank Adaptation (Hu et al. 2021) Key insight: weight updates during fine-tuning are LOW RANK Instead of updating W ∈ ℝᵈˣᵈ: W' = W + ΔW = W + B·A where B ∈ ℝᵈˣʳ, A ∈ ℝʳˣᵈ, r ≪ d If d=4096, r=8: ΔW has 4096² = 16M params B·A has 2×4096×8 = 65K params → 250x fewer! Only train A and B, keep W frozen. QLoRA (Dettmers et al. 2023): also quantize base model to 4-bit → Fine-tune 65B model on a single A100! // RLHF: Reinforcement Learning from Human Feedback (InstructGPT) Step 1: Supervised Fine-Tuning (SFT) Fine-tune pretrained LM on (prompt, ideal response) pairs L_SFT = -(1/T) Σₜ log π_SFT(yₜ | y<t, x) Step 2: Train a Reward Model Humans rank responses: (preferred yᵥ, rejected yₗ) L_RM = -log σ(r(x, yᵥ) - r(x, yₗ)) Step 3: PPO Optimization L_RLHF = E[r(x,y)] - β · KL(π_RL || π_SFT) KL term prevents model drifting too far from SFT // DPO: Direct Preference Optimization (Rafailov et al. 2023) Skip reward model entirely — optimize preferences directly: L_DPO = -E[log σ(β log(π_θ(yᵥ|x)/π_ref(yᵥ|x)) - β log(π_θ(yₗ|x)/π_ref(yₗ|x)))] → Simpler, more stable than RLHF, widely adopted → Used to align LLaMA 2 Chat, Zephyr, Tulu 2 // GRPO (Group Relative Policy Optimization — DeepSeek-R1, 2025) Samples G outputs per prompt, uses relative reward as baseline: Â_i = (rᵢ - mean(r)) / std(r) (no separate critic model needed) → Powers the reasoning capabilities of DeepSeek-R1

5.3 — RAG, Agents, and Modern LLM Systems

// RAG: Retrieval-Augmented Generation (Lewis et al. 2020) Problem: LLMs "hallucinate" facts and have knowledge cutoff Solution: retrieve relevant documents at inference time Architecture: 1. Query → Embedding (via encoder like BERT/E5/BGE) 2. Search vector database (Faiss, Pinecone, Weaviate, Chroma) score(q, dᵢ) = cosine_sim(embed(q), embed(dᵢ)) 3. Top-k docs → Context → LLM → Answer Dense Passage Retrieval (DPR): sim(q,d) = E_q(q)ᵀ · E_d(d) (separate encoders for query and doc) Fine-tuned with in-batch negatives: L = -log [exp(sim(q,d⁺)) / (exp(sim(q,d⁺)) + Σⱼ exp(sim(q,dⱼ⁻)))] // Agents: LLMs that can take actions Core loop: Observe → Think (chain-of-thought) → Act → Observe → ... ReAct (Yao et al. 2022): Interleave Reasoning and Acting: "Thought: I need to search for X" "Action: search(X)" "Observation: [result]" "Thought: Based on result..." Tool use: LLM outputs structured JSON for tool calls Function calling (OpenAI) / tool_use (Anthropic) APIs // Speculative Decoding — faster inference Small draft model proposes k tokens Large model verifies all k in parallel (single forward pass) Accept accepted tokens, reject rest Speedup: 2–3x with no quality loss

§ 7 — Research Level

Phase 6: Research Mastery

You now have the tools. The question is: how do you enter the research community, contribute original work, and push the field forward?

Phase 6

From Practitioner to Researcher

⏱ Duration: Ongoing — this phase never ends

Research is not a destination but a practice. It is reading, reproducing, breaking, fixing, questioning, and writing. Every great researcher failed a hundred experiments before one worked.

6.1 — How to Read Research Papers

  1. First pass (5 min): Title, Abstract, ConclusionUnderstand what they claim. Can you state the contribution in one sentence? If not, read again.
  2. Second pass (30 min): Introduction, Figures, Method section headersUnderstand the problem setting, why prior work fails, and at a high level what they propose.
  3. Third pass (2–3 hours): Full paper with equationsDerive every equation. Reproduce every table mentally. Ask: What assumptions are made? What would happen if assumption X were violated?
  4. Fourth pass: Reproduce the paperImplement their method from scratch (NOT using their code). If your numbers do not match, you do not understand the paper yet.
  5. Fifth pass: Critique and extendWhat are the limitations? What experiments were NOT run? What would you try differently?

6.2 — Active Research Areas (2024–2026)

Reasoning & RL for LLMs

Chain-of-thought, process reward models, MCTS for language, DeepSeek-R1 style GRPO training. Open frontier.

Efficient Architectures

Mamba (SSMs), RWKV, Hyena — replacing attention with linear-time alternatives. Rich theoretical depth.

Multimodal Models

Vision-language models (LLaVA, Flamingo), audio-language (Whisper+), video understanding. Fast moving.

Long Context

RoPE scaling, ALiBi, YaRN — extending context windows from 4K to 1M+ tokens. Active research.

Mechanistic Interpretability

Understanding WHAT and WHERE LLMs store knowledge. Circuits, superposition, feature geometry. Anthropic's core focus.

Quantization & Compression

GPTQ, AWQ, SmoothQuant, 1-bit LLMs (BitNet). Making LLMs run on edge devices. High practical impact.

Alignment & Safety

Constitutional AI, scalable oversight, reward modeling robustness. Critical as models get more capable.

Synthetic Data & Self-improvement

Distillation, self-play, model-generated training data. Phi series, Gemma: quality over quantity.

Mixture of Experts (MoE)

Sparse activation: only k of N expert FFN layers fire per token. Mixtral, DeepSeek-MoE, Grok-1.

6.3 — The Research Workflow

  1. Survey the literatureUse Semantic Scholar, arXiv, Papers With Code. For any area, read the 10 most cited papers from last 3 years first.
  2. Find the gapAfter reading 20+ papers: what question keeps being assumed away? What baseline is unfairly weak? What setting is unexplored?
  3. Formulate a hypothesisState it precisely. "We hypothesize that X improves Y on Z because of mechanism M." This forces clarity before you write code.
  4. Run ablations, not just the main experimentWhat happens if you remove component A? Replace B with C? The ablations are where understanding lives.
  5. Write continuouslyKeep a daily research log. Thoughts, experiments, failures. Your paper draft starts from your log.
  6. Submit, get rejected, revise, resubmitEvery great researcher has a drawer full of rejected papers. Rejection is data, not verdict.

§ 10 — Hands On

Project Roadmap

Reading without building is philosophy. You must get your hands dirty. Here is the sequence of projects that solidify each phase.

Month 3–4

NumPy Neural Network from Scratch

Implement forward pass, backpropagation, and training loop using ONLY NumPy. Train on MNIST. No PyTorch. No TensorFlow. This is non-negotiable. You will debug your understanding of the chain rule in the process.

Month 5–6

Micrograd & MLP on PyTorch

Reproduce Karpathy's micrograd autograd engine. Then rebuild as a full MLP in PyTorch. Implement and compare SGD, Adam, RMSProp. Understand what .backward() actually does in the computation graph.

Month 7–8

Word2Vec from Scratch

Implement skip-gram with negative sampling in pure PyTorch. Train on a Wikipedia dump. Visualize your embeddings with t-SNE. Verify king - man + woman ≈ queen in your embedding space.

Month 9–11

GPT from Scratch (nanoGPT-style)

Build a character-level or BPE-tokenized GPT. Implement multi-head attention, positional encoding, layer norm, residual connections from scratch. Train on Shakespeare. Then train a tiny version on your own text corpus.

Month 12–13

Fine-tune LLaMA with LoRA

Use HuggingFace + PEFT library to fine-tune a 7B or 13B LLaMA model on a custom dataset using LoRA. Understand adapter placement, rank choices. Evaluate with perplexity and human evaluation.

Month 14–15

Build a RAG Pipeline from Scratch

Implement document chunking, embedding with a sentence transformer, vector search with Faiss, retrieval, and generation with an open LLM. No LangChain shortcuts — build every component yourself first.

Month 16–18

Reproduce a Paper

Pick a 2022–2024 paper from an area you care about. Reproduce their main results without looking at their code. Write a blog post documenting your findings. Submit to arXiv. This is your entry into research.

Month 18+

Original Research Contribution

Identify a gap from your paper reading. Formulate a hypothesis. Run experiments. Write it up in NeurIPS/ACL/ICLR paper format. Submit even if rejected — the review feedback is invaluable mentorship.

§ 12 — From Your Professor

A Final Word on the Journey

The most important thing I can tell you Talent in this field is not a fixed thing. The researchers at the top of AI did not start that way. What separates them is one thing: they did not stop when it got hard. And it will get very hard. Phase 0 will feel like you are learning a new language. Phase 2 will feel like staring at an equation that refuses to make sense. This is normal. This is the process. The confusion is not a sign that you cannot do it. It is a sign that you are learning.

Daily Habits of Researchers

  • Read 1 paper per day (skim many, deep-read weekly)
  • Write code every single day, even small experiments
  • Keep a research journal (observations, failures, ideas)
  • Join reading groups, ML communities (MLSys Slack, EleutherAI, HuggingFace)
  • Teach what you learn — blog posts, Twitter threads, talks
  • Follow active researchers on arXiv, X, LinkedIn

Conferences to Target

  • NeurIPS — top ML venue, extremely competitive
  • ICML — International Conference on Machine Learning
  • ICLR — deep learning, representation learning
  • ACL / EMNLP / NAACL — NLP conferences

  • Workshops — easier to publish, great networking
